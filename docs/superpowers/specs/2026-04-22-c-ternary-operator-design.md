# C Frontend: Ternary Operator Design

## Context
The C language frontend already contains a `lower_ternary` function in `interpreter/frontends/c/expressions.py` registered to the `conditional_expression` node type. However, it is missing from the test coverage, causing `TERNARY_OPERATOR` to appear in the feature gaps report.

## Design

We will add comprehensive test coverage to validate the existing implementation and close the coverage gap.

### 1. Unit Tests
Add unit tests to `tests/unit/test_c_frontend.py`:
- **Basic Ternary (`test_c_ternary_operator`):**
  - Use the `@covers(CFeature.TERNARY_OPERATOR)` annotation.
  - Parse a simple C function containing a ternary expression (e.g., `int x = a ? b : c;`).
  - Assert that the CFG structure generated by `lower_ternary` is correct (contains a `BranchIf` instruction, and branches for `ternary_true`, `ternary_false`, and a merge block `ternary_end`).
- **Nested Ternary (`test_c_nested_ternary_operator`):**
  - Use the `@covers(CFeature.TERNARY_OPERATOR)` annotation.
  - Parse a C function with a nested ternary expression (e.g., `int x = a ? (b ? c : d) : e;`).
  - Assert that the CFG contains multiple `BranchIf` instructions and the correct sequence of nested branches and merge blocks.

### 2. Integration Test
Add a new end-to-end integration test `test_c_ternary_operator_execution` to `tests/unit/test_c_frontend.py`:
- Use the `@covers(CFeature.TERNARY_OPERATOR)` annotation.
- Provide a complete, runnable C program that uses both basic and nested ternary operators to compute results. For example:
  ```c
  int main() {
      int condition1 = 1;
      int condition2 = 0;
      int result1 = condition1 ? 42 : 99;
      int result2 = condition2 ? 10 : (condition1 ? 20 : 30);
      return result1 + result2;
  }
  ```
- Use `load_program` and `run_to_end` to execute the program in the VM.
- Assert that the VM reaches a `COMPLETED` state and the return value is the expected result (e.g., `42 + 20 = 62`).
