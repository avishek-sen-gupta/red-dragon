# C Frontend: Ternary Operator Design

## Context
The C language frontend already contains a `lower_ternary` function in `interpreter/frontends/c/expressions.py` registered to the `conditional_expression` node type. However, it is missing from the test coverage, causing `TERNARY_OPERATOR` to appear in the feature gaps report.

## Design

We will add comprehensive test coverage to validate the existing implementation and close the coverage gap.

### 1. Unit Test
Add a new unit test `test_c_ternary_operator` to `tests/unit/test_c_frontend.py`:
- Use the `@covers(CFeature.TERNARY_OPERATOR)` annotation.
- Parse a simple C function containing a ternary expression (e.g., `int x = a ? b : c;`).
- Assert that the CFG structure generated by `lower_ternary` is correct:
  - Contains a `BranchIf` instruction.
  - Contains branches for `ternary_true`, `ternary_false`, and a merge block `ternary_end`.

### 2. Integration Test
Add a new end-to-end integration test `test_c_ternary_operator_execution` to `tests/unit/test_c_frontend.py` (or a dedicated integration file if appropriate):
- Use the `@covers(CFeature.TERNARY_OPERATOR)` annotation.
- Provide a complete, runnable C program that uses the ternary operator to compute a result and return it from `main`. For example:
  ```c
  int main() {
      int condition = 1;
      int result = condition ? 42 : 99;
      return result;
  }
  ```
- Use `load_program` and `run_to_end` to execute the program in the VM.
- Assert that the VM reaches a `COMPLETED` state and the return value is the expected result (e.g., `42`). Include a test for the `false` condition as well to ensure both branches work correctly.
